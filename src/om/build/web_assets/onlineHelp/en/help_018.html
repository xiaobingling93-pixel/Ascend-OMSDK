Alarm Information

Scenarios

When a peripheral managed by the system is faulty, an alarm is reported to the Atlas IES. O&M personnel can obtain alarm information from the alarm page and rectify faults in a timely manner, improving O&M efficiency.

Parameter Description

Table 1 Parameters in the current alarms list

Parameter

Description

Alarm ID

Identifies an alarm of the Atlas 500 A2 edge station.

Severity

Indicates the alarm severity, which can be

Critical, Major, or Minor.

  • A critical alarm indicates a fault that may power off a device or even interrupt system services. Corrective actions must be taken immediately.

  • A major alarm will be generated when services are interrupted. This type of alarm has great impact on the system.

  • A minor alarm has a minor impact on the system, but you need to take corrective actions as soon as possible to prevent a more severe alarm.

Alarm Object

Indicates the device that generates an alarm.

Alarm Name

Describes an alarm.

Generated At

Indicates the time when an alarm is generated.
